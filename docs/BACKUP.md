# Backup

Guia rapido para proteger os dados do sistema antes de mudancas importantes.

## Quando fazer backup

- antes de grandes alteracoes;
- antes de importar notas de corretagem;
- antes de mexer em Firebase;
- antes de testar correcoes de risco;
- antes de publicar uma nova versao.

## Como fazer backup

- usar a funcao de exportacao do proprio sistema;
- salvar uma copia local em local seguro;
- manter pelo menos uma copia anterior conhecida como boa.

## Boas praticas

- nao editar o backup manualmente sem necessidade;
- conferir se o arquivo exportado abriu corretamente;
- manter backups separados por data;
- nunca seguir para uma alteracao arriscada sem copia recente.
